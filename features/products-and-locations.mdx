## Product Management

### Product Data Model

Products are stored in the `Productos` table:

| Field | Type | Description |
|-------|------|-------------|
| **Id** | int | Primary key |
| **Nombre** | nvarchar(100) | Product name (unique, required) |
| **Descripcion** | nvarchar(500) | Detailed description |
| **Codigo** | nvarchar(50) | Product code/SKU |
| **Categoria** | nvarchar(50) | Product category |
| **Estado** | nvarchar(20) | Activo or Inactivo |
| **FechaCreacion** | datetime | Creation timestamp |
| **FechaModificacion** | datetime | Last modified timestamp |

### Product Categories

DitzlerTotes supports product categorization:

- **Dulces** - Sweet syrups and confections
- **Aceites** - Oils and liquid fats
- **Jarabes** - Flavored syrups
- **Esencias** - Flavor essences and extracts
- **Otros** - Other products

<Tip>
Use consistent categories across your organization for better reporting and inventory management.
</Tip>

### Create Product

Add a new product to the catalog:

```bash
curl -X POST http://localhost:3000/api/admin/productos \
  -H "Content-Type: application/json" \
  -d '{
    "action": "create",
    "productData": {
      "nombre": "Aceite de Coco Refinado",
      "descripcion": "Aceite de coco refinado para uso industrial",
      "codigo": "ACR-003",
      "categoria": "Aceites",
      "estado": "Activo"
    }
  }'
```

<Steps>
  <Step title="Validate Name">
    Check that product name is unique and not empty
  </Step>
  <Step title="Generate Code">
    If no code provided, auto-generate from category prefix
  </Step>
  <Step title="Insert Record">
    Create product in Productos table
  </Step>
  <Step title="Audit Log">
    Record creation event with product details
  </Step>
</Steps>

### Update Product

Modify product information:

```bash
curl -X POST http://localhost:3000/api/admin/productos \
  -H "Content-Type: application/json" \
  -d '{
    "action": "update",
    "productData": {
      "id": 1,
      "nombre": "Jarabe de Chocolate Premium",
      "descripcion": "Jarabe de chocolate belga premium",
      "codigo": "JRC-001",
      "categoria": "Jarabes",
      "estado": "Activo"
    }
  }'
```

### Delete Product

Remove a product from the catalog:

```bash
curl -X POST http://localhost:3000/api/admin/productos \
  -H "Content-Type: application/json" \
  -d '{
    "action": "delete",
    "productData": {
      "id": 3
    }
  }'
```

<Warning>
Deleting a product will fail if there are totes currently using it. Set product status to "Inactivo" instead.
</Warning>

## Location Management

### Location Data Model

Locations are stored in the `Ubicaciones` table:

| Field | Type | Description |
|-------|------|-------------|
| **Id** | int | Primary key |
| **Nombre** | nvarchar(100) | Location name (unique, required) |
| **Descripcion** | nvarchar(500) | Detailed description |
| **Codigo** | nvarchar(50) | Location code |
| **Tipo** | nvarchar(50) | Location type |
| **Capacidad** | int | Maximum tote capacity |
| **Estado** | nvarchar(20) | Activo or Inactivo |
| **FechaCreacion** | datetime | Creation timestamp |
| **FechaModificacion** | datetime | Last modified timestamp |

### Location Types

Locations can be categorized by type:

- **Almacén** - General warehouse storage
- **Producción** - Production line areas
- **Despacho** - Shipping and dispatch zones
- **Mantenimiento** - Maintenance and cleaning areas
- **Residuos** - Waste disposal areas
- **Reproceso** - Reprocessing zones
- **Lavado** - Washing stations
- **Tránsito** - Temporary transit areas

## Capacity Management

### Tracking Capacity

The system tracks location capacity and utilization:

```javascript
// Calculated fields
{
  "Capacidad": 500,           // Maximum totes
  "TotesActuales": 342,       // Current tote count
  "Disponibilidad": 158,       // Available space (500 - 342)
  "Utilizacion": 68.4          // Utilization % (342/500 * 100)
}
```

### Capacity Alerts

The dashboard shows capacity warnings:

- **Green**: Less than 70% utilized
- **Yellow**: 70-90% utilized  
- **Red**: Over 90% utilized
- **Critical**: At or over 100% capacity

<Info>
Locations can temporarily exceed capacity during peak operations, but this should trigger investigation.
</Info>

## Integration with Totes

### Product Assignment

When filling a tote, the product is selected from the catalog:

```javascript
// Filling operator workflow
{
  "codigo": "TOTE-1234",
  "producto": "Jarabe de Chocolate",  // From Productos table
  "lote": "LOTE-240115",
  "peso": 285.5
}
```

### Location Assignment

Totes are always assigned to a location:

```javascript
// Location tracking
{
  "codigo": "TOTE-1234",
  "ubicacion": "Warehouse-A",  // From Ubicaciones table
  "estado": "LLENO"
}
```

### Movement Validation

When moving totes, the system validates:

1. Target location exists and is active
2. Target location has available capacity
3. Movement is allowed based on current tote state
4. User has permission to move to that location

## Reporting

### Product Usage Report

Track which products are most used:

```sql
SELECT 
  p.Nombre as Producto,
  COUNT(t.Id) as TotesActuales,
  SUM(t.Peso) as PesoTotal,
  AVG(t.Peso) as PesoPromedio
FROM Productos p
LEFT JOIN Totes t ON t.Producto = p.Nombre AND t.Activo = 1
GROUP BY p.Nombre
ORDER BY TotesActuales DESC;
```

### Location Utilization Report

Analyze warehouse utilization:

```sql
SELECT 
  u.Nombre as Ubicacion,
  u.Capacidad,
  COUNT(t.Id) as TotesActuales,
  CAST(COUNT(t.Id) * 100.0 / u.Capacidad AS DECIMAL(5,2)) as Utilizacion
FROM Ubicaciones u
LEFT JOIN Totes t ON t.Ubicacion = u.Nombre AND t.Activo = 1
WHERE u.Estado = 'Activo'
GROUP BY u.Nombre, u.Capacidad
ORDER BY Utilizacion DESC;
```
