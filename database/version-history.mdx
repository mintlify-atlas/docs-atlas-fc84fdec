## Version History

### v3.1 (January 2026) - Consolidation Update

**Release Date**: 2026-01-27

**Type**: Major schema update and consolidation

#### Changes

**New Tables**
- Added comprehensive `Productos` table with Ingredientes and Clientes columns
- Added comprehensive `Ubicaciones` table with Area and ClientesAsignados columns
- Standardized `Llenados_Totes` table structure

**Schema Updates - Totes**
- ✅ Added `CodigoRetail` column for retail tracking codes
- ✅ Added `UsuarioModificacion` column for audit trail
- ✅ Added `FechaRetorno` column (auto-calculated as FechaDespacho + 30 days)
- ✅ Renamed `Operador` column to `Usuario` for consistency
- ❌ Removed `FechaVencimiento` column (moved to Productos table)
- ❌ Removed `FechaEnvasado` column (no longer needed)

**Schema Updates - Clientes**
- ✅ Added `UbicacionDefecto` column for default client locations

**Schema Updates - Productos**
- ✅ Added `Ingredientes` column (NVARCHAR(MAX)) for ingredient lists
- ✅ Added `Clientes` column (NVARCHAR(MAX)) for client associations
- ✅ Added `FechaVencimiento` column (moved from Totes)

**Schema Updates - Ubicaciones**
- ✅ Added `Area` column (default: 'General')
- ✅ Added `ClientesAsignados` column for multi-client assignments

**Schema Updates - Llenados_Totes**
- ✅ Added `Sincronizado` column (BIT) for sync status tracking
- ✅ Added `CodigoBarra` column for barcode integration
- ✅ Renamed `Operador` to `Usuario` for consistency
- ✅ Extended `RFID_Tag` to NVARCHAR(100)
- ✅ Extended `Producto` to NVARCHAR(255)

**New Functions**
- `FN_GenerarCodigoSeguimiento()` - Generates tracking codes (COD-XXXXXX format)
- Enhanced `FN_GenerarCodigoTote()` - Improved tote code generation
- Maintained `FN_DiasHastaVencimiento()` - Days until expiration calculator

**Stored Procedures**
- Updated `SP_EstadisticasDashboard` to handle removed FechaVencimiento
- Updated `SP_CrearTote` to support new FechaRetorno logic
- Maintained all 6 core stored procedures

**Data Seeding**
- Added 9 operational locations (Almacén, Mantenimiento, Despacho, etc.)
- Created default users for all roles
- Updated admin credentials

#### Migration Requirements

<Warning>
  This version requires running two migration scripts in sequence.
</Warning>

```sql
-- 1. Execute master installation (for new installs)
USE master;
EXEC('source/docs/DitzlerDB_Master.sql');

-- 2. For existing databases, run migrations:
-- Migration 1: Move FechaVencimiento
USE Ditzler;
EXEC('source/docs/migrate_fecha_vencimiento.sql');

-- Migration 2: Add FechaRetorno
USE Ditzler;
EXEC('source/docs/add_fecha_retorno.sql');
```

#### Breaking Changes

- ⚠️ **FechaVencimiento removed from Totes** - Now stored at product level
- ⚠️ **FechaEnvasado removed from Totes** - No longer tracked
- ⚠️ **Column renamed**: Operador → Usuario (in Totes and Llenados_Totes)
- ⚠️ **API changes**: Endpoints expecting Totes.FechaVencimiento must be updated

#### Impact

**Tables Modified**: 5 (Totes, Productos, Clientes, Ubicaciones, Llenados_Totes)

**Stored Procedures Updated**: 2 (SP_EstadisticasDashboard, SP_CrearTote)

**Indexes Dropped**: 2 (IX_Totes_FechaVencimiento, IX_Totes_Dashboard)

**Data Migration**: Automatic for FechaRetorno; manual review needed for FechaVencimiento

```sql
-- FN_GenerarCodigoTote
-- Generates unique tote codes in format TOTE-0001
CREATE FUNCTION FN_GenerarCodigoTote()
RETURNS NVARCHAR(20)
AS
BEGIN
    DECLARE @codigo NVARCHAR(20);
    DECLARE @contador INT;
    SELECT @contador = ISNULL(MAX(CAST(SUBSTRING(Codigo, 6, LEN(Codigo)-5) AS INT)), 0) + 1
    FROM Totes 
    WHERE Codigo LIKE 'TOTE-%' 
    AND ISNUMERIC(SUBSTRING(Codigo, 6, LEN(Codigo)-5)) = 1;
    SET @codigo = 'TOTE-' + RIGHT('0000' + CAST(@contador AS NVARCHAR), 4);
    RETURN @codigo;
END
```
