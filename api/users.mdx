## User Management

Unified endpoint for all user management operations. Requires admin authentication.

<CodeGroup>
```bash POST /api/admin/users
curl -X POST https://api.ditzlertotes.com/api/admin/users \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "list"
  }'
```
</CodeGroup>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from login (must be an admin user)
</ParamField>

### Request Body

<ParamField body="action" type="string" required>
  Action to perform: `list`, `create`, `update`, or `delete`
</ParamField>

<ParamField body="userData" type="object">
  User data object (required for create, update, and delete actions)
</ParamField>

## Create User

Create a new user in the system.

<CodeGroup>
```bash POST /api/admin/users
curl -X POST https://api.ditzlertotes.com/api/admin/users \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "create",
    "userData": {
      "nombre": "Carlos",
      "apellido": "Ruiz",
      "email": "carlos.ruiz@example.com",
      "password": "securepassword123",
      "rol": "Operador"
    }
  }'
```
</CodeGroup>

### Request Body

<ParamField body="action" type="string" required>
  Must be `"create"`
</ParamField>

<ParamField body="userData" type="object" required>
  User data object
</ParamField>

<ParamField body="userData.nombre" type="string" required>
  User's first name
</ParamField>

<ParamField body="userData.apellido" type="string" required>
  User's last name
</ParamField>

<ParamField body="userData.email" type="string" required>
  User's email address (must be unique and valid format)
</ParamField>

<ParamField body="userData.password" type="string" required>
  User's password (minimum 6 characters)
</ParamField>

<ParamField body="userData.rol" type="string">
  User's role (defaults to "Operador" if not specified)
</ParamField>

### Response

<ResponseField name="success" type="boolean">

</ResponseField>

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

### Response Examples

<CodeGroup>
```json Success Response

```


## Important Notes

### Root Admin Protection

The root admin account (`admin@ditzler.com`) has special protections:

- Cannot be deleted
- Must always maintain admin role ("Administrador" or "Admin")
- Cannot have the "usuarios" function excluded from preferences

### Email Validation

- Email addresses must be in valid format
- Email addresses must be unique across all users
- Changing a user's email is validated against existing users

### Password Requirements

- Minimum 6 characters
- Password is optional when updating (only updated if provided and not empty)

### Audit Logging

All user management operations (create, update, delete) are automatically logged to the audit system with:

- Current user information
- Action performed
- Old and new values (for updates)
- Timestamp and IP address

### Error Handling

Common error messages:

- `"No autorizado"` - Missing or invalid authentication token
- `"Datos de usuario requeridos"` - Missing userData in request
- `"ID de usuario requerido"` - Missing user ID for update/delete
- `"Acción no válida"` - Invalid action parameter
- `"El formato del email no es válido"` - Email format validation failed
- `"Ya existe un usuario con ese email"` - Duplicate email constraint violation
- `"Usuario no encontrado"` - User ID does not exist
- `"Los datos no cumplen con las validaciones requeridas"` - Database constraint violation
