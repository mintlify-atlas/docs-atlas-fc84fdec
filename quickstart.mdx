This guide will help you set up and run DitzlerTotes in the shortest time possible. Follow these steps to get your development environment ready.

## Prerequisites

Before starting, ensure you have:

- **Node.js** version 18 or higher
- **SQL Server** 2016 or higher (Express edition works fine)
- **npm** package manager
- **Git** for cloning the repository

<Tip>
You can check your Node.js version with `node --version`. If you need to upgrade, download the latest LTS version from [nodejs.org](https://nodejs.org).
</Tip>

## Installation

<Steps>

<Step title="Clone the repository">

First, clone the DitzlerTotes repository to your local machine:

```bash
git clone <repository-url>
cd DitzlerTotesDB
```

</Step>

<Step title="Install dependencies">

Install all required Node.js packages:

```bash
npm install
```

This will install:
- Express 5.1.0 (web framework)
- mssql 11.0.1 (SQL Server client)
- bcrypt 6.0.0 (password hashing)
- helmet 8.1.0 (security middleware)
- express-rate-limit 8.2.1 (rate limiting)
- express-session 1.18.2 (session management)
- dotenv 17.2.3 (environment variables)
- jsonwebtoken 9.0.3 (JWT handling)

</Step>

<Step title="Configure environment variables">

Create your environment configuration file:

```bash
cp .env.example .env
```

Then edit `.env` with your database credentials:

```env .env
# Database Configuration
DB_USER=your_database_user
DB_PASSWORD=your_database_password
DB_DATABASE=Ditzler
DB_SERVER=localhost
DB_PORT=1433

# Server Configuration
PORT=3003
NODE_ENV=development

# Session Security
SESSION_SECRET=change-this-to-a-random-secret-key-in-production
```

<Warning>
Never commit your `.env` file to version control. It contains sensitive credentials.
</Warning>

</Step>

<Step title="Generate a secure session secret">

For production use, generate a strong random secret:

<CodeGroup>

```bash Node.js
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```

```powershell PowerShell
[guid]::NewGuid().ToString() + [guid]::NewGuid().ToString()
```

</CodeGroup>

Copy the output and update `SESSION_SECRET` in your `.env` file.

</Step>

<Step title="Set up the database">

Execute the database installation scripts in SQL Server. You have two options:

**Option 1: Master installation script**

```sql
-- Run database/00_Master_Install.sql in SQL Server Management Studio
```

**Option 2: Run scripts sequentially**

```sql
-- Execute in order:
-- 1. database/01_Usuarios_Table.sql
-- 2. database/02_Clientes_Table.sql
-- 3. database/03_Totes_Table.sql
-- 4. database/04_Eventos_Table.sql
-- 5. database/05_Update_User_Roles.sql
-- 6. database/06_Add_Llenado_Role.sql
-- 7. database/07_Multi_Role_Support.sql
-- 8. database/08_User_Preferences.sql
-- 9. database/09_Productos_Ubicaciones.sql
```

This creates:
- 6 main tables (Usuarios, Clientes, Totes, Llenados_Totes, Eventos, Productos, Ubicaciones)
- 18+ stored procedures
- 3 database functions
- 39+ indexes for performance

</Step>

<Step title="Start the server">

Now you're ready to launch DitzlerTotes:

<CodeGroup>

```bash Production
npm start
```

```bash Development (with auto-reload)
npm run dev
```

</CodeGroup>

You should see:

```
🚀 Servidor DitzlerTotesDB ejecutándose en http://localhost:3003
📦 Arquitectura modular activa (Fase 1 completada)
📁 Módulos migrados: Auth, Totes, Users, Clients, Products, Locations
```

</Step>

<Step title="Verify the installation">

Test the database connection:

```bash
curl http://localhost:3003/api/test-db-connection
```

Test the audit system:

```bash
curl http://localhost:3003/api/test-audit
```

Both endpoints should return successful responses.

</Step>

</Steps>

## Running tests

DitzlerTotes includes a test suite built with Jest:

```bash
npm test
```

The test configuration is already set up in `package.json` with `jsdom` environment for frontend testing.

## Development mode

For active development, use nodemon for automatic server restarts:

```bash
npm run dev
```

This will watch for changes and reload the server automatically.

<Note>
The server includes anti-cache middleware for development that prevents browser caching of JavaScript and CSS files. You'll always get the latest versions during development.
</Note>

## Common configuration

The frontend configuration is located in `js/config.js`:

```javascript js/config.js
const CONFIG = {
  SESSION: {
    STORAGE_KEY: 'loggedInAdmin',
    TIMEOUT: 8 * 60 * 60 * 1000  // 8 hours
  },
  API: {
    BASE_URL: '',
    TIMEOUT: 10000
  },
  UI: {
    ITEMS_PER_PAGE: 50,
    REFRESH_INTERVAL: 30000
  }
};
```

You can adjust these values to match your requirements.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database connection fails">
    Check your SQL Server credentials in `.env`. Ensure SQL Server is running and accepting connections on the specified port (default 1433).
  </Accordion>
  
  <Accordion title="Port already in use">
    Change the `PORT` value in your `.env` file to an available port, such as 3003 or 3005.
  </Accordion>
  
  <Accordion title="Login fails with correct credentials">
    Verify that the database scripts executed successfully and the Usuarios table contains the default accounts. Check the server logs for authentication errors.
  </Accordion>
  
  <Accordion title="Session expires immediately">
    Ensure `SESSION_SECRET` is set in your `.env` file. The session timeout is configured to 8 hours by default in `js/config.js`.
  </Accordion>
</AccordionGroup>
